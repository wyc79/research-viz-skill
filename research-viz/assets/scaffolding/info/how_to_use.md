# How to use this folder

This workspace was generated by the `research-viz` skill. Three things you can do, all from the project root (the directory that contains `data/` and `visualizations/`).

## 1. Parse / clean raw data

```bash
bash visualizations/parse_input.sh
```

What it does: walks the sibling `data/` directory, reads each tabular file (csv, tsv, txt, xlsx), runs three quality checks (per-column dtype consistency, missing-value counts, format anomalies), prompts you (interactively) for a missing-data strategy per column, and writes the cleaned outputs into `visualizations/intermediate_data/`.

**Output layout** (depends on how many input files there are):

- **One input file** → a single `intermediate_data/parsed_results.csv` plus `parsed_results.meta.json`.
- **Many input files** (default `--combine per_file`) → the parser **mirrors `data/`'s folder structure under `intermediate_data/`**, writing one cleaned CSV per input. Useful for research data where each file is one session / patient / client / experiment and shouldn't be aggregated. There is intentionally **no** `parsed_results.csv` in this mode.
- **Force a single combined frame** (with a `__source__` column tagging each row's origin): pass `--combine concat` or `--combine both` (writes both layouts).

`intermediate_data/parsed_index.json` is **always** produced — it lists every per-file output (rows, dtypes, strategies applied) and whether a combined CSV exists. Downstream tools and future agents read this to discover what's there.

To rerun non-interactively with a known strategy:

```bash
bash visualizations/parse_input.sh --no-interactive \
    --strategy '{"temperature_C":"median","comment":"drop_col"}'
```

Available strategies per column: `ignore`, `drop_row`, `drop_col`, `mean`, `median`, `mode`, `ffill`, `bfill`, `constant:<value>`.

## 2. Generate a static plot

```bash
bash visualizations/generate_plot.sh "<your plot request in plain English>"
```

Examples:

```bash
bash visualizations/generate_plot.sh "scatter of mass vs luminosity colored by spectral_class, log y"
bash visualizations/generate_plot.sh "violin per treatment group of recovery_time, faceted by sex"
bash visualizations/generate_plot.sh "correlation heatmap of all numeric columns"
```

Output: `plots/<slug>/figure.png` (300dpi) plus the underlying tidy CSV used to build the figure (so the plot is reproducible). The slug is derived from your prompt; you can also pass `--slug myname` to override.

In **per-file** mode (no combined `parsed_results.csv`), pass `--data <path-to-specific-csv>` to point at a single per-file output, e.g.:

```bash
bash visualizations/generate_plot.sh "metric over time" \
    --data visualizations/intermediate_data/patient_001/session_a.csv \
    --slug patient_001-session_a-metric
```

If you call `generate_plot.sh` without `--data` and no combined CSV exists, the script lists the available per-file outputs and asks you to pick.

## 3. Interactive streamlit explorer

```bash
bash visualizations/interactive_page.sh
```

Launches `streamlit run visualizations/streamlit/index.py` and opens it in your browser. New exploration topics live in `visualizations/streamlit/pages/*.py` and show up automatically in the sidebar.

In per-file mode the landing page shows a sidebar selector for switching between parsed files. If your dataset is very large (>1M rows or >200MB) and live charts feel slow, the agent may switch a page to a "viewer" that displays pre-rendered PNGs from `plots/<slug>/` — check `info/context.md` for whether that's the case.

## Running on a new / different / fuller data folder

The wrappers default to the sibling `data/` directory, but every step honors the `DATA_DIR` env var (and `--data-dir` for the parser). Two common workflows:

### (a) Pilot first, full data later

This is the recommended path for large or per-session research datasets:

```bash
# 1. Curate a small representative subset alongside your full data
mkdir -p pilot_data
cp data/patient_{001,007,019}/*.csv pilot_data/    # whatever shape "representative" means for you

# 2. Iterate cheaply on the pilot
DATA_DIR="$(pwd)/pilot_data" bash visualizations/parse_input.sh --no-interactive
DATA_DIR="$(pwd)/pilot_data" bash visualizations/generate_plot.sh "violin of metric per session"
DATA_DIR="$(pwd)/pilot_data" bash visualizations/interactive_page.sh   # tweak filters until happy

# 3. When the visuals are right, switch to the full data — no script edits required
DATA_DIR="$(pwd)/data" bash visualizations/parse_input.sh --no-interactive --strategy '{...}'
DATA_DIR="$(pwd)/data" bash visualizations/generate_plot.sh "violin of metric per session"
```

When you swap `DATA_DIR`, the parser overwrites `intermediate_data/` with the new run's outputs (a fresh `parsed_index.json`, fresh per-file mirrors). Your plots and streamlit pages reread automatically.

### (b) Pointing at a sibling dataset elsewhere on disk

```bash
DATA_DIR=/path/to/some/other/dataset bash visualizations/parse_input.sh
```

Or for a one-off plot only:

```bash
bash visualizations/generate_plot.sh "scatter of x vs y" \
    --data /path/to/some/other/parsed.csv
```

## Adding a new visualization later

If you're working with a fresh agent, just point it at this folder and ask for what you want. The skill is designed to read `info/context.md` first to figure out what's already been done. If you've changed files yourself outside the agent loop, mention that — or let the agent reconcile by glancing at `scripts/` and `plots/`.
