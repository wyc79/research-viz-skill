# visualizations/

Sensor-log streamlit workspace, generated by the `research-viz` skill on 2026-05-04 and trimmed to a single-file streamlit-only project.

The sibling `data/` folder (with `sensor_log.csv`) is treated as **read-only** — nothing here will modify, rename, or delete files in `data/`. All transformed outputs land in `intermediate_data/`.

See [`info/how_to_use.md`](info/how_to_use.md) for the full human guide.

## Two entry points

- `parse_input.sh` — read `data/sensor_log.csv`, parse `ts` to datetime, sort, write `intermediate_data/sensor_log__parsed.csv` + `parsed_index.json`.
- `interactive_page.sh` — launch the streamlit dashboard. Sidebar filters by `site` and `sensor`; the time-series page plots `value` over time with one line per (site, sensor) combination.

## Quickstart

```bash
bash visualizations/parse_input.sh
bash visualizations/interactive_page.sh
```

Intermediate files follow the `<original_dataset_name>__<stage>.csv` convention (here: `sensor_log__parsed.csv`).
