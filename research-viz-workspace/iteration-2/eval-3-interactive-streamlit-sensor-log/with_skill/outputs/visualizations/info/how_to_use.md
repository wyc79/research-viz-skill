# How to use this folder

This workspace was generated by the `research-viz` skill. Three things you can do, all from the project root (the directory that contains `data/` and `visualizations/`).

## 1. Parse / clean raw data

```bash
bash visualizations/parse_input.sh
```

What it does: walks `/Users/yuanchenwang/Documents/Claude/Projects/Visualization Skill/research-viz-workspace/iteration-2/eval-3-interactive-streamlit-sensor-log/with_skill/outputs/data`, reads each tabular file (csv, tsv, txt, xlsx), runs three quality checks (per-column dtype consistency, missing-value counts, format anomalies), prompts you (interactively) for a missing-data strategy per column, and writes:

- `visualizations/intermediate_data/parsed_results.csv` — the cleaned dataset.
- `visualizations/intermediate_data/parsed_results.meta.json` — what was done and to which columns.

To rerun non-interactively with a known strategy:

```bash
bash visualizations/parse_input.sh --strategy '{"temperature_C":"median","comment":"drop_col"}'
```

Available strategies per column: `ignore`, `drop_row`, `drop_col`, `mean`, `median`, `mode`, `ffill`, `bfill`, `constant:<value>`.

## 2. Generate a static plot

```bash
bash visualizations/generate_plot.sh "<your plot request in plain English>"
```

Examples:

```bash
bash visualizations/generate_plot.sh "scatter of mass vs luminosity colored by spectral_class, log y"
bash visualizations/generate_plot.sh "violin per treatment group of recovery_time, faceted by sex"
bash visualizations/generate_plot.sh "correlation heatmap of all numeric columns"
```

Output: `plots/<slug>/figure.png` (300dpi) plus the underlying tidy CSV used to build the figure (so the plot is reproducible). The slug is derived from your prompt; you can also pass `--slug myname` to override.

## 3. Interactive streamlit explorer

```bash
bash visualizations/interactive_page.sh
```

Launches `streamlit run visualizations/streamlit/index.py` and opens it in your browser. New exploration topics live in `visualizations/streamlit/pages/*.py` and show up automatically in the sidebar.

If your dataset is very large (>1M rows or >200MB) and the live charts feel slow, the agent may switch a page to a "viewer" that displays pre-rendered PNGs from `plots/<slug>/` — check `info/context.md` for whether that's the case.
